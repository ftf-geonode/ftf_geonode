APIs

Introduction

Developers can interact with the Data Driven Farming GeoNode through 3 sets of APIs: CSW, OGC Services, and the internal GeoNode API.

CSW

The CSW endpoint is: /catalogue/csw. To retrieve metadata records run something like this:

/catalogue/csw?request=GetRecords&service=CSW&version=2.0.2&resultType=results&maxRecords=1000&elementsetname=full&typenames=csw:Record&outputschema=http://www.isotc211.org/2005/gmd.

This endpoint can be used by other systems, such as Data.Gov and HDX, to automatically harvest records from the Data Driven Farming GeoNode.

OGC

All layers in the Data Driven Farming GeoNode are accessible via OGC Services. The WMS Capabilities document is accessible with: /geoserver/wms?SERVICE=WMS&REQUEST=GetCapabilities. This endpoint can be used by geospatial analysts to pull layers directly into their desktop environment (/docs/integration.html). OGC services can also be used to pull layers directly from the Data Driven Farming GeoNode into web maps to make mashups.

Internal

In addition to CSW and OGC APIs, GeoNode also has an internal API for retrieving a list of layers, publishing data, and more. You can review the community opensource code at https://github.com/geonode/geonode. If you have questions, get in contact on the geonode-devel listserve.
